## 二、后端修改内容

## 2.1 新增当前用户头像上传接口

修改文件：

- `bootstrap/src/main/java/com/nageoffer/ai/ragent/user/controller/UserController.java`
- `bootstrap/src/main/java/com/nageoffer/ai/ragent/user/service/UserService.java`
- `bootstrap/src/main/java/com/nageoffer/ai/ragent/user/service/impl/UserServiceImpl.java`

新增接口：

```http
POST /api/ragent/user/avatar
Content-Type: multipart/form-data
Authorization: <token>
```

请求参数：

```text
file: 头像图片文件
```

返回数据：

```json
{
  "userId": "当前用户ID",
  "username": "当前用户名",
  "role": "当前用户角色",
  "avatar": "/api/ragent/uploads/avatars/xxx.png"
}
```

核心逻辑：

- 校验文件不能为空。
- 限制文件大小不超过 `5MB`。
- 限制文件扩展名为 `jpg`、`jpeg`、`png`、`gif`、`webp`。
- 校验 `Content-Type` 必须以 `image/` 开头。
- 文件保存到本地目录：

```text
uploads/avatars/
```

- 数据库存储头像访问路径：

```text
/api/ragent/uploads/avatars/{userId}-{uuid}.{ext}
```

- 上传新头像后会尝试删除旧的本地头像文件，旧文件删除失败不影响新头像生效。

## 2.2 新增本地头像静态资源访问映射

修改文件：

- `bootstrap/src/main/java/com/nageoffer/ai/ragent/rag/config/WebConfig.java`

新增资源映射：

```java
registry.addResourceHandler("/uploads/**")
        .addResourceLocations(uploadLocation);
```

效果：

- 后端可以直接访问本地上传后的头像文件。
- 访问路径示例：

```text
/api/ragent/uploads/avatars/123-abc.png
```

## 2.3 放行头像静态资源访问

修改文件：

- `bootstrap/src/main/java/com/nageoffer/ai/ragent/user/config/SaTokenConfig.java`

调整内容：

- `SaInterceptor`
- `DemoModeInterceptor`
- `UserContextInterceptor`

均新增排除路径：

```java
.excludePathPatterns("/auth/**", "/uploads/**", "/error")
```

原因：

- 浏览器 `<img>` 加载头像时不会自动携带前端 Axios 的 `Authorization` 请求头。
- 因此头像静态资源必须允许匿名读取，否则头像上传成功后页面无法展示。



同步逻辑：

- 聊天侧边栏和后台顶部头像随即刷新。

## 七、后续建议

- 修复本机 JDK 环境，将 `JAVA_HOME` 指向可用 JDK 17 后重新执行 Maven 编译。
- 如需更严格安全策略，可以在头像上传后增加图片重编码，防止伪装图片或 EXIF 敏感信息残留。
- 如需生产环境部署，可以把本地 `uploads/avatars` 替换为 RustFS/S3，并保留当前接口协议不变。
- 如需让头像资源也鉴权访问，可以改为后端代理读取接口，但前端 `<img>` 需要配合带 token 的 blob 加载方式。
