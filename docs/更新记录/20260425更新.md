# 20260425 更新记录

## 1. 任务目标

本次修改的目标是：**彻底移除 pipeline / DAG 编排能力在当前产品主链路中的代码入口、前端入口、后端实现和初始化 SQL 定义**，将文档处理流程统一收敛为：

- 文档上传
- Tika 解析
- chunk 策略切分
- embedding 向量化
- 向量库写入

同时要求：

- 删除后端 pipeline 实现与前端配置入口
- 保证现有知识库文档链路仍可运行
- 保证编译通过、前端构建通过
- 输出清晰的操作记录与测试记录

## 2. 操作记录

### 2.1 代码审计与范围确认

首先对仓库中的 pipeline 相关实现进行了全量排查，确认当前功能散落在以下几层：

- 后端 `bootstrap/src/main/java/com/nageoffer/ai/ragent/ingestion/**`
- 文档上传与分块主服务 `KnowledgeDocumentServiceImpl`
- 知识库文档请求/返回 DTO 与页面参数
- 前端管理台 `frontend/src/pages/admin/ingestion/IngestionPage.tsx`
- 前端 `frontend/src/services/ingestionService.ts`
- 路由与后台菜单
- 数据库初始化脚本中的 `t_ingestion_*` 表及 `process_mode / pipeline_id` 字段

结论：如果只删页面入口，不删服务实现和 schema，后续维护仍然会被误导，因此本次按“**入口 + 实现 + schema + 测试**”一并清理。

### 2.2 后端：移除 pipeline 主体实现

已删除整个 pipeline 实现包中的核心能力，包括但不限于：

- 控制器：`ingestion/controller/**`
- 请求/返回对象：`ingestion/controller/request/**`、`ingestion/controller/vo/**`
- 实体与 Mapper：`ingestion/dao/**`
- DAG 上下文、结果与配置模型：`ingestion/domain/**`
- 编排引擎：`ingestion/engine/**`
- 节点实现：`ingestion/node/**`
- Prompt 与工具类：`ingestion/prompt/**`、`ingestion/util/**`
- 服务实现：`ingestion/service/IngestionPipelineService.java`
- 服务实现：`ingestion/service/IngestionTaskService.java`
- 具体实现：`ingestion/service/impl/IngestionPipelineServiceImpl.java`
- 具体实现：`ingestion/service/impl/IngestionTaskServiceImpl.java`
- 抓取器策略：`ingestion/strategy/fetcher/**`

删除后，pipeline 的创建、编辑、执行、任务跟踪等能力已经不再存在于后端主代码路径。

### 2.3 后端：保留文档主链路，统一收敛为 chunk-only

已将知识库文档处理逻辑统一收敛为单一路径：

- 文件/URL 上传后，文档只进入 chunk 模式
- 不再支持通过 `processMode=pipeline` 进入 DAG 编排
- 更新文档时也不再允许切换 pipeline
- 分块日志只保留 chunk 维度字段

  - 分块日志不再记录 pipeline 信息

- `bootstrap/src/main/java/com/nageoffer/ai/ragent/knowledge/controller/request/KnowledgeDocumentUploadRequest.java`

- `bootstrap/src/main/java/com/nageoffer/ai/ragent/knowledge/controller/vo/KnowledgeDocumentChunkLogVO.java`

### 2.4 后端：清理遗留依赖并做包归位

在删掉 pipeline 模块后，发现有两个遗留问题同步处理：

#### 2.4.1 远程 URL 文档抓取能力不能跟着 pipeline 一起删

`RemoteFileFetcher` 仍然需要 HTTP 拉取能力，否则 URL 上传和定时刷新会坏掉。

因此没有简单删除 `HttpClientHelper`，而是进行了能力迁移：

- 新增：`bootstrap/src/main/java/com/nageoffer/ai/ragent/knowledge/handler/HttpClientHelper.java`
- `RemoteFileFetcher` 改为依赖 knowledge 模块内的新 helper

这一步的目的是：**保留 URL 文档能力，删除 pipeline 私有依赖**。

#### 2.4.2 `IntentTreeService` 原来挂在 `ingestion` 包下，语义已经不合适

`IntentTreeService` 本身属于意图树管理，不是 pipeline 功能。

因此本次做了包归位：

- 原路径迁移到：`bootstrap/src/main/java/com/nageoffer/ai/ragent/rag/service/IntentTreeService.java`
- 原实现迁移到：`bootstrap/src/main/java/com/nageoffer/ai/ragent/rag/service/impl/IntentTreeServiceImpl.java`

同步修改引用：

- `bootstrap/src/main/java/com/nageoffer/ai/ragent/rag/controller/IntentTreeController.java`
- `bootstrap/src/test/java/com/nageoffer/ai/ragent/rag/Intent/IntentTreeServiceTests.java`

处理完成后，`com.nageoffer.ai.ragent.ingestion` 包已不再保留运行期代码。
