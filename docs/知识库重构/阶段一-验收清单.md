## 手动验收项

1. 文档上传
   - 上传弹窗可以选择 `通用文档 / 项目报告 / 学术论文`
   - 不选择特殊类型时默认保存为 `general`
   - 上传完成后文档列表能看到文档类型

2. 文档编辑
   - 文档详情/编辑弹窗可以修改文档类型
   - 保存后刷新列表，`docType` 不丢失

3. 分块任务
   - 启动分块后，文档列表能展示实际 `parseEngine / chunkEngine`
   - 分块日志能展示本次任务的 `docType / parseEngine / chunkEngine`
   - 失败日志也会保留已经解析出的引擎信息，便于排障

4. 向量 metadata
   - 每个 chunk metadata 应包含 `doc_type / source_type / ingest_source_type`
   - 每个 chunk metadata 应包含 `parse_engine / parser_type / chunk_engine / chunk_fallback`
   - 重新启用文档触发向量重建时，metadata 不应丢失

5. 聊天检索
   - 聊天输入区可以选择 `全部文档 / 通用文档 / 项目报告 / 学术论文`
   - 选择具体类型后，请求会带 `docType`
   - 后端会把 `docType` 转成向量检索的 `metadataFilters.doc_type`
   - 选择 `全部文档` 时保持原有混合检索行为

6. 数据库迁移
   - 新库使用 `resources/database/schema_pg.sql`
   - 老库从 v1.2 升级可执行 `resources/database/upgrade_v1.2_to_v1.3_doc_type_router.sql`
   - 已经处于 v1.3 的环境可执行 `resources/database/upgrade_v1.3_to_v1.4_stage1_observability.sql`
