## 手动验收项

1. 默认策略
   - 新上传文档默认切块策略为 `qmd_smart`
   - 前端上传/编辑弹窗能看到 QMD 参数：`maxChars / overlapChars / windowChars`
   - 仍可手动选择 `fixed_size / structure_aware` 用于灰度或回退

2. QMD 脚本
   - 在 `scripts/qmd` 下执行 `npm install`
   - 执行 `npm run smoke`
   - 输出应为 JSON，且包含 `engine: "qmd"` 和非空 `chunks`

3. 后端切块链路
   - `qmd_smart` 会进入 `QmdDocumentChunkEngine`
   - QMD 成功时实际 `chunkEngine` 为 `qmd`
   - QMD 失败、超时或返回空结果时回退到 `structure_aware`
   - 回退时分块任务不应整体失败

4. 可观测字段
   - 文档表 `chunk_engine` 可看到最终切块引擎
   - 分块日志 `chunk_engine` 可看到本次任务实际切块引擎
   - 向量 metadata 包含 `chunk_strategy / chunk_engine / chunk_fallback`
   - QMD 回退时 `chunk_fallback = true`

5. 协议安全
   - QMD 适配脚本必须返回 `engine = "qmd"`
   - chunks 为空、文本为空或进程失败时，Java 侧会拒绝结果
   - 大量 stderr 输出不会阻塞 Java 侧等待
